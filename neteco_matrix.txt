Overview				
	This document describes the ports to be opened for the NetEco to achieve proper service communication and provides the design workflow of the OM communication ports to guide engineers to set firewalls. Do not open ports that are not mentioned in this document on the firewalls.

Intended Audience				
	"This document is intended for:  
●Technical support engineers
●Maintenance engineers

Structure				
	Describe the content of each sheet and the meaning of each field in port tables. 			
	Description of each sheet			
				
	Ports Design Process	Describes the firewall setting principles, port design process, and precautions. Read this sheet before you set a fire wall. 		
	Ports Between the NetEco and the NMS 	Describes the communication ports between the NetEco server and the NMS devices. The NMS devices include third-party management devices (such as the SMS server) provided by some telecom operators. 		
	Ports Opened by the NetEco for NEs 	Describes ports to be opened by the NetEco server for each NE. If the source device is an NE, the port needs to be opened for all NEs that access to the NetEco system. If the specific situation has been specified for some source devices or in special scenarios, the port only needs to be opened for these NEs. 		
	OM Ports to Be Opened for UMTS, GSM, and SingleRAN 	Describes ports to be opened for UMTS, GSM, and SingleRAN. 		
	Ports Between the NetEco and Client	Describes ports, including the ports opened by the NetEco server for the web-based NetEco client and other OM clients, to be opened by the NetEco server for the client. 		
	Ports on the Hardware Server	Open ports on a hardware server, including ports that may be opened on the TaiShan rack server. Do not open these ports on the firewall.		
	MAE	The ports that are opened in all networking modes include the ports between the MAE server and the NetEco server, the ports between MAE and NEs.		
	Third-party Tool 	In the NetEco networking system, some functions, such as the client remote access and system backup, require the use of third-party tools. If the firewalls are deployed between the third-party tools and other devices, the ports must be opened by following the instructions provided in this sheet.		
	Appendix - Local Ports	Describes local ports on the NetEco server and ports that do not need to be opened on the firewall. These ports are used on the internal network and will be scanned during the port scanning.		
				
	Description of each field in port tables			
	Source Device	"Port for sending signaling traffic, that is, the start point where signaling traffic is sent to Destination Port Number of Target Device. Generally, the number of the port used by Source Device is randomly selected from 1024 to 65535. If the source device uses a specified port, the information about the specified port is added in Port Description.
If Source Device is Local, it indicates that a local port is used as the source device. In such a case, the port does not need to be opened on the firewall."		
	Source IP Address	IP address of source device.		
	Source Port	Source Port: indicates the port from which the signal flow is initiated, that is, the signal flow is transmitted from this port of a source device to the destination port of a destination device."1024–65535" indicates that the source port is a random port within this range. 		
	Destination Device	"Port for receiving signaling traffic, which needs to be opened for Source Device, that is Destination Port Number. If Target Device is Local, it indicates
that a local port is used as the destination device. In such a case, the port does not need to be opened on the firewall."		
	Destination IP Address	IP address of target device.		
	Destination Port (Listening)	Indicates the listening port that or a number segment in which the ports can provide services on the server. If the destination port contains two or more ports, and one of the ports contains special illustration, for example, 31100, 31600 (TLS), port 31100 is a non-encryption port and port 31600 is an TLS encryption port. This situation is also specified in the port description. You need to open related ports based on the actual transmission mode. If the destination port contains two or more ports, and the port difference is not specified in the port description, you need to open all these ports. 		
	Protocol	Protocols used at the transport layer, such as TCP, UDP, and SCTP.		
	Port Description	Description of a port, which mainly includes the functions that are implemented through the port.		
	Listening Port Configurable (Y/N)	Indicates whether the listening port can be changed on the web UI or the CLI. 		
	Authentication Mode	Authentication mode of a port. If the value is None, it indicates that no authentication is performed.		
	Encryption Mode	Encryption mode of a port. If the value is None, it indicates that no encryption is performed.		
	Plane	Describes which plane the listening port belongs to, such as one of the OM plane, control/signaling plane, and user plane, or all the three planes. For the NetEco, all ports belong to the OM plane. 		
	Version	"Indicates the versions of products to which a port is applicable. 
If one or multiple version numbers are available, it indicates that the port is applicable only to the version or versions.
If all versions are available, it indicates that the port is applicable to all versions.
If the port is applicable to a version and later versions or a version and earlier versions, this information will be added."		
	Special Scenario	"Special scenario to which a port is only applicable. For example, a port is applicable only to a certain type of NE, a certain networking scenario, or a specified type of server.
If the value is None, it indicates that the port is a common port and applicable to all scenarios."		
				
OM Port Designing Procedure			
1	Confirm the positions for deploying firewalls, and then design the OM ports to be opened on the firewalls.		
	Firewalls are generally deployed in the following positions:		
	"● Between the NetEco server and network management system (NMS) servers
● Between the NetEco server and NEs
● Between the NetEco server and clients
● Between the Citrix client and Citrix server (applicable only when the Citrix tool is used)"		
2	Collect field system information, including the information about third-party tools, northbound interfaces, NEs, and special scenarios, and then determine the ports to be opened according to actual requirements.		
	"NOTICE: 
1) You need to enable the function of detecting the FTP transfer on firewalls to ensure that data is properly transmitted through FTP. For details about how to enable this function, see the documents related to the actual firewall.
2) The source IP addresses of all OM ports (on the NE side) opened for OSS servers such as the U2000, NetEco, and AP Manager servers contain the physical IP addresses and floating IP addresses configured for the service Ethernet ports of these OSS servers. "
